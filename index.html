Mybatis概述

mybatis是一个机遇java的持久层框架，它内部封装了jdbc，使开发者只需要关注sql语句本身，而不需要花费时间来处理和加载驱动、创建连接、创建statment等繁琐的过程.。它使用ORM思想实现了结果集的封装。

MyBatis可以使用简单的XML或注解用于配置和原始映射，将接口和Java的POJO（Plain Old Java Objects，普通的Java对象）映射成数据库中的记录。

Redis使用

发表于 2020-03-27 分类于 redis

操作关系型数据库的数据非常耗时，需要建立连接、读取数据。但是在缓存中的数据可以直接使用，速度快，于是就有了将数据库储存在缓存中的技术，将一些不太经常变化的数据储存在缓存中。使用的时候回从缓存中查询，如果没有，就从数据库读取出来存入缓存，这样在下次访问这个数据的时候，可以直接从缓存中拿出来，不必再去访问数据库。由于把数据存入缓存中使用，数据可能会存在很久不会改变，所以存入一些不需要经常改变的数据，就可以提高访问速度。

ssm环境搭建

发表于 2020-03-23 更新于 2020-03-28 分类于 spring框架

前言

ssm框架，是spring + Spring MVC + MyBatis的缩写，这个是继SSH之后，目前比较主流的Java EE企业级框架，适用于搭建各种大型的企业级应用系统。越来越多的公司选择使用ssm轻量级框架组合来开发JavaEE项目，掌握这一框架无疑会给开发者带来更强的竞争力。

大致过程：当客户端发起请求时，会先到达DispatcherServlet（前端控制器，其实就是一个Servlet），前端控制器根据HandlerMapping（处理器映射器）查找url对应的Handler，之后再调用HandlerAdapter（处理器适配器）来执行对应的Hanlder，这时候Handler会返回一个带有逻辑视图以及模型数据的ModelAndView给HandlerAdpater并再次返回给DispatcherServlet，前端控制器调用视图解析器对ModelAndView进行视图渲染，并且向前端相应处理结果。

本篇文章讲的是如何去搭建一个ssm应用，本文使用的是eclipse。
